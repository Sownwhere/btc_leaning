# `btc_sma20_backtest.py` 策略原理介绍与结果分析

## 1) 策略原理介绍（SMA20 趋势跟随）

这个脚本实现的是一个非常经典的**均线择时**策略，核心思想是：

- 当价格在均线上方，认为市场偏强，持仓；
- 当价格在均线下方，认为市场偏弱，空仓；
- 通过“只在趋势明确时参与”来降低无效交易。

对应到 `btc_sma20_backtest.py` 的计算流程：

1. 下载 BTC 历史数据  
   使用 `yfinance` 获取从 2020-01-01 开始的日线数据。

2. 计算日收益率 `ret`  
   `ret_t = Close_t / Close_{t-1} - 1`  
   用于表示“如果当天持仓，能赚/亏多少”。

3. 计算 20 日均线 `signal`  
   `signal_t = mean(Close_{t-19} ... Close_t)`  
   作为趋势判断基准。

4. 生成仓位 `position`  
   - `Close > SMA20` -> `position = 1`（持仓）
   - 否则 `position = 0`（空仓）

5. 计算策略收益 `strategy`  
   `strategy_t = position_{t-1} * ret_t`  
   这里用了 `shift(1)`，代表“今天用昨天决定的仓位”，避免前视偏差（未来函数）。

6. 计算策略净值 `equity` 与基准净值 `buy_hold`  
   - `equity = cumprod(1 + strategy)`  
   - `buy_hold = cumprod(1 + ret)`  
   两者对比可以判断择时是否优于长期持有。

## 2) 图表怎么看（你运行脚本后会看到两张图）

### 上图：价格 + SMA20

- 价格线在均线上方时，策略大概率处于持仓状态；
- 价格线反复穿越均线时，会出现频繁切换，容易被“震荡行情”来回打脸。

### 下图：策略净值 vs 买入持有

- 蓝线（策略）向上更平滑：说明回撤控制可能更好；
- 橙线（买入持有）更陡但波动大：说明收益和风险都更高；
- 若蓝线长期低于橙线：说明该参数下择时没有跑赢持有。

## 3) 结果分析（适合新手的结论模板）

你可以按下面结构写自己的回测结论：

1. **收益对比**  
   先看最终净值谁更高：`equity` vs `buy_hold`。

2. **风险对比**  
   观察谁的回撤更深、恢复更慢。  
   如果策略收益略低但回撤显著更小，依然有实战价值。

3. **交易频率判断**  
   SMA20 较短，常在震荡期频繁开平仓；真实交易中会被手续费明显侵蚀。

4. **市场适配性**  
   BTC 是长期强趋势且高波动资产，Buy & Hold 常常是很强基准。  
   均线择时更常见目标是“降低回撤”，而不是一定“跑赢收益”。

---

## 4) 当前 `btc_sma20_backtest.py` 的局限

- 只用单一信号（SMA20），对震荡行情不够鲁棒；
- 暂未显式统计指标（如年化收益、最大回撤、夏普比率）；
- 未计入手续费与滑点，结果偏乐观；
- 参数未做系统化测试（例如 SMA50/SMA100/SMA200）。

---

## 5) 下一步优化建议（可直接落地）

- 拉长均线周期：优先测试 `SMA50`、`SMA100`、`SMA200`；
- 加过滤器：如 `ATR` 或 `ADX`，仅在趋势更明确时交易；
- 加交易成本：至少加入单边/双边手续费；
- 输出指标表：总收益、年化收益、最大回撤、胜率、交易次数；
- 做参数对比：同一时期内对比多组参数，避免只看单次回测。

---

## 6) 一句话总结

`btc_sma20_backtest.py` 是一个非常好的入门策略：结构清晰、逻辑正确（已避免前视偏差），但要用于更真实的评估，必须加入过滤器、交易成本和风险指标。
